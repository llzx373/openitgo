## 1. 如何启用 `use_columns`

CSS Columns 分页器目前通过运行时功能开关 `window.ebookUseColumns` 控制，**默认关闭**，旧的行盒分页器保持完整。

1. 打开任意 EPUB / TXT / MOBI / Markdown 电子书。
2. 通过阅读器设置或 Rust 调试入口将 `use_columns` 设为 `true`：
   - 在 `ebook_renderer.rs` 发送给 WebView 的设置 JSON 中设置 `"use_columns": true`。
   - 或在 DevTools 控制台执行 `applySettings(JSON.stringify({...currentSettings, use_columns: true}))`。
3. 重新加载章节后，JS 会使用 `#column-view` / `#column-content` 进行 CSS Columns 分页，旧的 `#spread` / `#measure` 容器保留但隐藏。

### 关键观察点

- **下一页/上一页**：是否跨章节正常、是否在首页/末页循环、动画是否流畅。
- **调整窗口大小**：是否保留大致阅读位置，是否出现空白页或重复内容。
- **改字体/字号**：重新分页后是否回到原位置附近，是否触发除零/NaN 等异常。
- **目录跳转**：能否跳到指定章节和锚点；fragment-less href 是否按章节开头处理。
- **搜索高亮**：高亮标记是否显示、是否能前后切换、切换后是否自动滚动到匹配处。

## 3. 已知限制

1. **单页/双页模式下的超高图片**：若图片高度超过一页，`break-inside: avoid` 只能尽量保持完整；浏览器在必要时仍会拆分，可能出现图片跨页。
2. **宽表格**：表格宽度超过列宽时会被压缩或出现横向滚动条；双页模式下宽表格更容易溢出可视区域。
3. **EPUB 自定义 CSS 冲突**：部分 EPUB 自带 `column-*`、`position: fixed`、`transform` 等样式可能与注入的 column 样式冲突，导致排版错乱。
4. **连续滚动模式下的页数统计**：滚动模式按视口高度估算“页数”，与真实页数只是近似对应。
5. **搜索高亮在重排后丢失**：字体、字号、窗口大小变化会导致章节 DOM 重建，搜索高亮需要重新执行 `findText`。
6. **动画与快速翻页**：连续快速翻页会取消上一次动画定时器，但极低概率在动画过渡期间产生轻微抖动。

## 4. 收集真实 EPUB 与问题上报

1. **收集样本**：至少准备 10~20 本不同类型的 EPUB，覆盖小说、技术书、图文书、漫画、儿童绘本、杂志等。
2. **建立测试记录**：为每本书记录书名、来源、内容类型、测试结果、异常截图/录屏、触发步骤。
3. **最小复现步骤**：遇到 bug 时，先确认：
   - 是否仅在 `use_columns = true` 时出现；
   - 是否仅在特定模式（单页/双页/滚动）下出现；
   - 是否在旧分页器（`use_columns = false`）下同样出现。
4. **提交 issue 时提供**：
   - 书籍文件名或匿名化章节 HTML 片段；
   - 触发步骤与预期/实际结果；
   - 应用版本与 commit SHA；
   - 控制台日志或 IPC 调试信息（可通过 DevTools 抓取）。
